# Spec: Pause Should Not Replace Birds

**Date:** 2026-06-20
**Scope:** `src/hooks/useSoundscape.ts` and its tests only.

---

## Problem

`stopAll()` is called on both user-initiated pause and on rebuild (pin change). It runs `a.src = ''; a.load()` on every audio element, which fires `error` events. The error handler sees `retries < MAX_AUDIO_RETRIES`, increments the counter, and eventually exhausts retries â€” pulling spare voices from the pool. A second pause then destroys those spare birds. The voice list should only change when a new pin is dropped.

---

## Fix

### 1. Add `pauseAll()`

New `useCallback` alongside `stopAll`. Called from `toggle()` when the user pauses.

```typescript
const pauseAll = useCallback(() => {
  timersRef.current.forEach(t => clearTimeout(t));
  timersRef.current = [];
  audioRefs.current.forEach(a => { a.pause(); a.currentTime = 0; });
  isPlayingRef.current = false;
  setIsPlaying(false);
  setVoices(v => v.map(voice => ({ ...voice, isActive: false })));
}, []);
```

Key difference from `stopAll`: **no `a.src = ''` and no `a.load()`**. Audio elements remain loaded; no error events fire; retry/spare logic never triggers.

### 2. `stopAll()` unchanged â€” rebuild only

`stopAll()` keeps its current body (including `a.src = ''; a.load()`). It is only called from the rebuild `useEffect` when `recordingsKey`/`recentObsKey` change (i.e. when a new pin is dropped). It is **no longer called from `toggle()`**.

### 3. `toggle()` uses `pauseAll()`

```typescript
const toggle = useCallback(() => {
  if (isPlayingRef.current) {
    pauseAll();
  } else {
    // ... existing play logic unchanged
  }
}, [pauseAll, startVoice]);
```

`stopAll` is removed from `toggle`'s dependency array; `pauseAll` is added.

### 4. Guard against duplicate `ended` listeners in `startVoice`

`startVoice` adds an `ended` listener with `{ once: true }` every time it is called. On resume after pause, `startVoice` is called again while the pre-pause listener is still pending (audio never reached `ended`). Two listeners would then both fire when the audio finishes.

Fix: add `pendingEndedRef = useRef<boolean[]>([])`. In `startVoice`, only add the `ended` listener if `!pendingEndedRef.current[index]`. Set it `true` when adding, `false` when the handler fires. Reset all entries to `false` in the rebuild effect (after `stopAll`).

```typescript
const pendingEndedRef = useRef<boolean[]>([]);

// In rebuild effect, after stopAll():
pendingEndedRef.current = [];

// In startVoice:
void audio.play();
setVoices(v => v.map((voice, i) => i === index ? { ...voice, isActive: true } : voice));

if (!pendingEndedRef.current[index]) {
  pendingEndedRef.current[index] = true;
  audio.addEventListener('ended', () => {
    pendingEndedRef.current[index] = false;
    setVoices(v => v.map((voice, i) => i === index ? { ...voice, isActive: false } : voice));
    if (!isPlayingRef.current) return;
    const delay = applyJitter(intervalsRef.current[index] ?? MAX_INTERVAL_MS);
    timersRef.current.push(setTimeout(() => startVoice(index), delay));
  }, { once: true } as AddEventListenerOptions);
}
```

## Tests

Update `src/hooks/useSoundscape.test.ts`:

- **Existing test** `'toggle() stop pauses audio and clears isActive'`: verify `voices` array length and `sciName` values are unchanged after pause (not just `isActive: false`)
- **New test**: after pause + resume cycle, no duplicate `ended` callbacks fire â€” verify `startVoice` is not called more than once per cycle

- **New test**: voice list is identical before and after a pause/resume cycle (same species, same count)

## Definition of Done

- Pausing never triggers spare pool replacement
- Voice list is identical before and after any number of pause/resume cycles
- Voice list only changes when `recordingsKey` or `recentObsKey` changes (new pin)
- `npm run build` clean, `npm test` passes
