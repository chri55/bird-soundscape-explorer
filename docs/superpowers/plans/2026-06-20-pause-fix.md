# Pause Fix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix pause so it only suspends playback ŌĆö voices never change until a new pin is dropped.

**Architecture:** Split `stopAll` (full teardown, rebuild-only) from a new `pauseAll` (suspend-only, no `src` clearing). Add `pendingEndedRef` to prevent duplicate `ended` listeners on resume. Both changes are in `useSoundscape.ts` only.

**Tech Stack:** React 19 + TypeScript, Vitest + React Testing Library.

## Global Constraints

- `verbatimModuleSyntax: true` ŌĆö all type-only imports must use `import type`
- Vitest `globals: true` ŌĆö no explicit `describe`/`it`/`expect`/`vi`/`beforeEach`/`afterEach` imports in test files
- Test command: `npx vitest run src/hooks/useSoundscape.test.ts` (single file), `npx vitest run` (full suite)
- Build command: `npm run build`
- `pauseAll` must NOT call `a.src = ''` or `a.load()` ŌĆö this is the core invariant
- `stopAll` keeps its current body unchanged ŌĆö still called only from the rebuild `useEffect`
- Voice list (`voices` state) must be identical before and after any pause/resume cycle
